# 自检工具使用指南

### 说明

接入过程中遇到的绝大数问题是由于APK包不完整或基本设置错误导致的，自检工具可以协助您在开发过程中自行检查

APK自检工具下载：[点击下载](https://pkg.biligame.com/tool/bili-open/tool/CP%E8%87%AA%E6%A3%80%E5%B7%A5%E5%85%B7_1_0_4.zip)  
提供功能：用于检查接入SDK的apk包，检查apk接入SDK的基本配置  
适用版本：适用于所有版本  
运行环境：windows/mac/linux + jdk

### 使用步骤

####1、在"检查项"选择游戏包已经接入的SDK

目前选项：

* 哔哩哔哩游戏SDK(biliSDK)+哔哩哔哩游戏数据SDK(biliDataSDK)

##### 注意：

* 请使用非加固包而且没有混淆SDK代码的包检查  
* 请先确认是否已经接入了对应选项中的SDK

####2、选择游戏母包路径及SDKDemo路径

根据上面选择的检查项，我们会看到对应检查项的界面，根据界面提示，输入或者点击后面的选择按钮，填入相应文件的路径

##### 注：选择SDK Demo对比包路径时，使用`Demo Apk`文件夹下普通安卓工程构建的demo包

##### 例如：`BSGameSDK_ Android_Demo.apk`，不要使用cocos2d、u3d demo包

####3、点击"执行检查",等待输出结果

1. 检查过程中界面会卡住，此为正常现象，请耐心等待输出结果
2. 执行检查时，请不要占用工具生成的srcApk、sdks文件夹及其子文件，以免检查失败
3. 根据每一项的检测结果提示进行修复，通常来说`警告`类提示可以不管，但是`错误`类提示则必须修复
